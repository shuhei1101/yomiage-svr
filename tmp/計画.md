# VoiceBox MCP - Ollama連携 大規模改修計画

## 改修の目的

クライアント側でのキャラクター指定を不要にし、サーバー側でOllamaを使ってテキストをキャラクターの口調に自動変換する仕組みを導入する。

---

## 現在の構成 vs 新しい構成

### 現在の構成

```
クライアント
  ↓ (text + speaker_name)
VoiceBox API Server
  ↓ (text + speaker_id)
VOICEVOX Engine
  ↓ (WAV audio)
音声再生
```

**問題点:**
- クライアント側でキャラクター（speaker_name）を明示的に指定する必要がある
- テキストがそのまま音声合成されるため、キャラクターらしさはVOICEVOXの声質のみ


### 新しい構成

```
クライアント
  ↓ (text のみ)
VoiceBox API Server
  ├─→ キャラクター選択ロジック
  │    ├─ ランダム選択
  │    ├─ 文脈ベース選択（オプション）
  │    └─ ラウンドロビン（オプション）
  ↓
Ollama Server
  ↓ (text + 口調プロンプト)
  ↓ テキスト変換（キャラクターの口調に調整）
  ↓ (変換後テキスト)
VoiceBox API Server
  ↓ (transformed_text + speaker_id)
VOICEVOX Engine
  ↓ (WAV audio)
音声再生
```

**改善点:**
- クライアントは生のテキストだけ送信（シンプル化）
- サーバー側で口調変換 → VOICEVOX という2段階処理
- キャラクターの個性がテキスト内容にも反映される

### 3. キャラクター選択ロジック

**ファイル:** `yomiage-svr/services/character_selector.py` (新規)

**責務:**
- テキストの文脈からキャラクターを選択
- ランダム選択・ラウンドロビン・ルールベース選択のサポート

**選択戦略:**

#### 戦略1: ランダム選択（初期実装）
```python
import random

class CharacterSelector:
    def select(self, text: str) -> str:
        """テキストに応じてキャラクターを選択"""
        return random.choice(list(SPEAKERS.keys()))
```

#### 戦略2: ラウンドロビン（オプション）
```python
class RoundRobinSelector:
    def __init__(self):
        self.index = 0
        self.speakers = list(SPEAKERS.keys())
    
    def select(self, text: str) -> str:
        speaker = self.speakers[self.index]
        self.index = (self.index + 1) % len(self.speakers)
        return speaker
```

#### 戦略3: ルールベース（将来拡張）
```python
class RuleBasedSelector:
    def select(self, text: str) -> str:
        """テキストの内容から適切なキャラクターを選択"""
        if "エラー" in text or "問題" in text or "警告" in text:
            return "metan"  # 緊張感のある報告
        elif "完了" in text or "成功" in text:
            return "zundamon"  # 元気な報告
        else:
            return "tsumugi"  # 丁寧な対応
```

#### 処理フロー

```python
@app.post("/speech", response_model=SpeechResponse)
async def create_speech(
    request: SpeechRequest,
    background_tasks: BackgroundTasks
):
    # speaker_nameが指定されていない場合は自動選択 + Ollama変換
    if request.speaker_name is None:
        # 1. キャラクター選択
        selector = CharacterSelector()
        speaker_name = selector.select(request.text)
        
        # 2. Ollamaでテキスト変換
        ollama = OllamaService()
        speaker = get_speaker(speaker_name)
        transformed_text = await ollama.transform_text(
            request.text,
            speaker.tone_prompt
        )
        
        # 3. 音声合成
        tool = SpeechTool()
        background_tasks.add_task(
            tool.speak,
            transformed_text,
            speaker_name
        )
        
        return SpeechResponse(
            message=f"音声合成を開始しました (話者: {speaker_name}, 変換後: {transformed_text[:30]}...)",
            speaker_name=speaker_name,
            original_text=request.text,
            transformed_text=transformed_text
        )
    
    # speaker_nameが指定されている場合は従来通り（Ollama変換なし）
    else:
        tool = SpeechTool()
        background_tasks.add_task(
            tool.speak,
            request.text,
            request.speaker_name
        )
        
        return SpeechResponse(
            message=f"音声合成を開始しました (話者: {request.speaker_name})",
            speaker_name=request.speaker_name,
            original_text=request.text,
            transformed_text=None  # 変換なし
        )
```

## リスクと対策

### リスク1: Ollama変換が遅い

**対策:**
- 軽量モデル使用（gemma2:2b、qwen2.5:3b）
- タイムアウト設定（10秒）
- 変換失敗時は元のテキストで音声合成にフォールバック

### リスク2: Ollamaサーバーが起動していない

**対策:**
- ヘルスチェック実装（起動時にOllama接続確認）
- 接続失敗時は警告ログ + 元のテキストで続行

### リスク3: プロンプトの品質が低い

**対策:**
- Few-shot例を充実させる
- 実際の変換結果を見てプロンプトを反復改善
- キャラクター別の口調ファイルを外部化（YAML/JSON）

### リスク4: 既存クライアントとの互換性

**対策:**
- `speaker_name` をオプションにすることで後方互換性維持
- 既存の明示的指定は従来通り動作
